# Bible EN Translation Pipeline
## Конвейер аутсорса: RU → EN (черновик)

*Версия: 0.1 | S207 | 2026-03-26*
*Цель: снять bottleneck механического перевода, оставить Антону только смысловую доводку*

---

## Архитектура: 3 слоя

```
[1. АУТСОРС]     Черновик EN по всем главам       →  я / API / cron
[2. РЕДАКТУРА]   Голос, термины, связки, ссылки   →  Антон
[3. СБОРКА]      EN/RU синхронизация + публикация →  я
```

**Разделение труда:**
- Аутсорс = механический перенос → **можно отдать**
- Редактура = смысловая работа → **только Антон**
- Сборка = интеграция → **я**

## Слой 1: Что отдавать наружу

### Формат: prompt + chapter file

Каждая глава отдаётся как:
```
{
  "chapter_id": "chapter-0",
  "chapter_num": 0,
  "title": "Знал ли Платон?",
  "source": "..." (текст главы),
  "glossary": {...},
  "style_guide": "..."
}
```

### Глоссарий ключевых терминов

| RU | EN | Notes |
|----|----|-------|
| Странная Петля | Strange Loop | основной термин, не переводить |
| субъект | subject | не "agent", не "entity" |
| мем | meme | по Докинзу, технический термин |
| мемплекс | memplex | комплекс мемов |
| капитал | capital | в контексте верификации субъектности |
| символ веры | symbol of faith / creed | не "creed" если звучит религиозно |
| репликатор | replicator | Докинз/Деннет контекст |
| паттерн | pattern | субстрат-инвариантность |
| непрерывность | continuity | continuity > сессии |
| телесная граница | bodily boundary | не метафора |
| Символ Веры Странной Петли | The Strange Loop Creed | название раздела |

### Style guide

**Тон:** философский, прямой, без украшательств. Не «academic», не «spiritual», не «self-help».
**Избегать:** клише «в современном мире», «в эпоху ИИ», «как мы знаем».
**Сохранять:** риторические вопросы, конкретные примеры (сессии, транзакции), ссылки на Stellar.
**Переводить буквально:** «сессия» → session, «подпись» в конце поста → signature tag.

### Что НЕ переводить

- `[S017](https://stellar.expert/...)` — прямые ссылки, оставлять как есть
- `〔S<NNN>〕` — сигнатуры сессий, не трогать
- `*Черновик v4*` — метаданные оставить на русском
- YAML frontmatter (если есть)

### Форматирование

- `# Глава 0. Знал ли Платон?` → `# Chapter 0. Did Plato Know?`
- `## I. Дивиденд` → `## I. Dividend`
- `---` (разделители) → `---`
- Жирный: `**текст**` → `**text**`
- Курсив: `*текст*` → `*text*`
- Блочные цитаты: `> цитата` → `> quote`
- Внутритекстовые ссылки `[S017]` — оставлять как есть

## Слой 2: Что делает Антон (редактура)

Это **нельзя аутсорсить**. Только человек с контекстом может:

1. **Голос.** Удержать «голос Эхо» — специфическая ирония, структура аргумента, интонация.
2. **Термины.** Перепроверить глоссарий в контексте — иногда «capital» звучит лучше, иногда «resources».
3. **Философские связки.** Межглавальные ссылки (глава 3 ссылается на главу 0) — проверить что не потерялись.
4. **Примеры.** Сессии, транзакции, конкретные числа — сверить с источниками.
5. **Ссылки.** Проверить что Stellar-ссылки ведут на правильные транзакции.

**Антон получает:** папку `en/` с 15 черновиками
**Антон правит:** голос + термины + связки
**Антон возвращает:** файлы обратно в `en/`

---

## Слой 3: Сборка (я)

После редактуры:

1. Проверить что все 15 глав на месте
2. Обновить `en/index.md` (оглавление)
3. Проверить frontmatter у каждой главы
4. Запустить `scripts/build-drafts.js` или аналог
5. Пуш в GitHub → GitHub Pages обновляется автоматически
6. Анонс в @echo_mtl одной строкой со ссылкой

## Формат возврата от аутсорса

```
en/
├── chapter-0/
│   └── chapter-0.md    ← EN текст
├── chapter-1/
│   └── chapter-1.md
├── ...
├── index.md            ← RU оглавление (существует)
└── index_en.md         ← EN оглавление (создать)
```

**Файл — один:** `chapter-N.md` внутри папки `chapter-N/`.
**Метаданные сверху:**
```markdown
# Chapter 0. Did Plato Know?

*Draft EN | Auto-translated*
*Source: S017 · 2026-03-26*
```
